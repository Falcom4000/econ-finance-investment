# 56.4 现金流估值、相对估值和网络估值的适用边界

来源：Marco Di Maggio, *Blockchain, Crypto and DeFi: Bridging Finance and Technology*, Chapters 1-4 and 9；补充参照本笔记第 33 章关于股票估值、第 38 章关于绩效度量的讨论。

加密资产估值困难，不是因为金融学失效，而是因为资产权利经常不清楚。贴现现金流模型要求有可归属现金流；相对估值要求有可比资产；网络估值要求用户和价值之间有稳定关系。很多 token 并不满足这些条件。

因此，估值方法不能机械套用。要先问资产类型，再选工具。

这和前面股票估值部分的原则完全一致。估值模型不是魔法公式，而是把未来经济利益折算到今天。若未来经济利益属于谁都不清楚，模型就只能制造精确幻觉。加密资产的难点，常常不是不会计算折现率，而是不知道 token 持有人到底能拿到什么。

## 现金流估值适用于谁

如果一个 token 持有人能分享协议现金流，现金流估值才有基础。例如，协议收取手续费，扣除运营和激励成本后，部分收入用于回购、分红或分配给质押者。此时，可以把 token 看成某种经济权益。

但很多治理代币没有明确现金流权。协议收入可能给 LP、验证者、开发团队或资金库，未必给 token 持有人。此时用市盈率或 DCF 估值会产生虚假精确。

现金流估值还要考虑监管。若向 token 持有人分配收入，可能更接近证券，面临更严格监管。许多项目故意避免明确分红，使估值更依赖未来治理预期。

这形成一个估值悖论：权利越清楚，越容易估值，也越可能触发证券监管；权利越模糊，监管压力可能暂时较小，但估值基础也更弱。投资者不能把“未来治理可能分配收入”直接当成当前现金流权，因为治理可能改变，监管可能阻止，团队和大户也可能选择不同分配方式。

## 相对估值的用途

相对估值比较同类资产，如公链市值/手续费、DEX 市值/交易量、借贷协议市值/收入、稳定币发行人估值/流通量。它的优点是简单，能反映市场如何给类似资产定价。

缺点是可比性差。两个公链的费用机制、发行、燃烧、Layer 2 分流、开发者生态和安全模型可能完全不同。两个 DEX 的交易量质量也可能不同，一个来自真实用户，一个来自补贴和刷量。

相对估值适合提出问题，不适合直接给出结论。若某协议估值远高于同类，需要解释它是否有更强增长、护城河、价值捕获或风险更低。

相对估值最有价值的地方，是迫使投资者解释差异。两个 DEX 都有 10 亿美元交易量，一个 token 市值是另一个的 5 倍，这并不直接说明谁贵谁便宜，而是提出问题：交易量质量是否不同？费用率是否不同？用户留存是否不同？治理代币是否有收入权？安全风险是否不同？如果回答不了这些问题，倍数比较就只是表面数字。

## 网络估值和 Metcalfe 直觉

网络估值常用一个直觉：网络价值随用户数量非线性增长。用户越多，连接越多，价值越高。这类似通信网络、社交网络和支付网络。

加密网络确实有网络效应，但用户数和价值之间关系不稳定。地址可以被拆分，交易可以被机器人刷出，用户质量差异很大。一个高净值机构地址和一个空投刷量地址不能简单等同。

因此，网络估值要用更高质量指标，如真实活跃用户、付费用户、费用收入、稳定币沉淀、开发者留存和流动性深度，而不是单纯地址数。

## 稀缺性估值的边界

BTC 常被用稀缺性估值。固定供给确实重要，但稀缺性不是价值充分条件。没有需求的稀缺物品仍然可能没有价值。

Stock-to-flow 等模型试图用存量和新增供给解释价格，但容易忽略需求变化、利率环境、风险偏好和监管冲击。供给曲线稳定，不代表需求曲线稳定。

稀缺性更适合作为长期叙事基础，而不是精确估值模型。投资者仍需分析采用率、持有人结构、流动性和宏观条件。

## RWA 和证券型 token 应回到传统估值

代币化国债、房地产、私募信贷和基金份额，不应该因为上链就换一套神秘估值。它们的价值来自底层现金流、信用风险、折现率、流动性和法律权利。

区块链可能带来流动性溢价改善、结算成本下降或可分割性提升，但底层资产质量仍是核心。若一个房地产项目租金差、产权不清、杠杆高，上链不能改变这些问题。

所以，RWA token 的估值要先按传统资产估值，再考虑链上结构带来的流动性、托管和监管调整。

## 小结

加密资产估值要根据资产权利选择方法。现金流估值适用于有明确费用分配或收益权的 token；相对估值可用于同类比较但要警惕可比性；网络估值强调用户和流动性网络效应，但必须使用高质量指标；稀缺性估值适合解释长期叙事，不能单独决定价格；RWA 和证券型 token 应回到底层现金流和法律权利。估值的核心不是找一个万能公式，而是确认价值如何被 token 捕获。
