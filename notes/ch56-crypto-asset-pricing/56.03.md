# 56.3 网络价值、用户增长和链上基本面指标

来源：Marco Di Maggio, *Blockchain, Crypto and DeFi: Bridging Finance and Technology*, Chapters 1-4 and 9；补充参照本笔记第 31 章关于因子和风险溢价、第 33 章关于股权估值的讨论。

传统股票有财务报表：收入、利润、资产、负债和现金流。加密协议没有完全相同的报表，但有链上数据。活跃地址、交易量、手续费、TVL、稳定币供应、开发者活动、质押率、流动性深度，都可能帮助判断网络使用情况。

这些指标常被称为链上基本面。但“能看到数据”不等于“数据一定代表价值”。链上指标需要解释，也可能被激励扭曲。

学习链上数据时，要把它当成一种新的会计材料，而不是自动真相。财务报表也需要分析收入质量、应收账款、一次性收益和会计政策；链上数据同样需要分析地址质量、交易动机、补贴来源和费用支付意愿。透明只是起点，解释才是投资分析。

## 网络效应和用户增长

许多加密资产依赖网络效应。使用者越多，网络越有价值；开发者越多，应用越多；应用越多，用户和流动性越多。Ethereum、Solana、稳定币网络和 DeFi 协议都具有这种特征。

网络效应可以带来正反馈。更多用户提高流动性，更多流动性吸引更多应用，更多应用又吸引更多用户。早期平台估值常常反映未来网络效应，而不是当前现金流。

但网络效应也可能反向运行。安全事故、监管打击、流动性撤出或补贴停止，会让用户离开，开发者减少，资产价格下跌，生态进一步萎缩。

## 常见链上指标

活跃地址衡量参与者数量，但一个用户可以创建多个地址，地址数可能被刷量。交易量衡量使用强度，但高频套利和机器人交易可能放大数字。TVL 衡量锁定资产规模，但会随币价上涨而机械上升，也可能由补贴吸引。

手续费收入更接近用户支付意愿。如果用户愿意持续支付费用，说明网络资源有真实需求。但高费用也可能伤害用户体验，并把活动推向 Layer 2 或竞争链。

稳定币供应和流动性深度反映金融活动基础。一个链如果没有足够稳定币、DEX 深度和借贷市场，很难支持复杂金融应用。

## 指标必须看生成机制

第 43 章讲回测时强调，数据生成机制非常重要。链上基本面也一样。

一个协议 TVL 很高，可能因为用户真实需要它，也可能因为项目发放代币补贴。补贴结束后，资金可能迅速离开。一个链交易量很高，可能来自真实支付，也可能来自低成本刷量。一个治理代币持有人很多，可能是社区广泛，也可能是空投导致的临时地址。

因此，分析链上数据要问：谁在使用？为什么使用？是否支付真实费用？补贴停止后是否留下？使用是否能转化为 token 价值？

可以按一个顺序阅读指标。先看用户是否真实存在，再看用户是否愿意付费，再看费用是否留在协议或 token，最后看这些收益是否超过安全、补贴和运营成本。只看中间某一个数字，比如 TVL 或交易量，很容易被短期激励误导。

## 从用户指标到价值捕获

网络被使用，不等于 token 捕获价值。稳定币交易量巨大，但收益可能被发行人而非公链 token 捕获。DeFi 协议交易量很大，但如果费用全部给 LP，治理代币未必受益。Layer 2 活跃度高，价值可能被排序器、应用或用户分走。

价值捕获要看费用路径和权利结构。协议收入归谁？费用是否燃烧？是否回购 token？治理能否改变分配？token 是否承担安全或抵押功能？

这和股票估值类似。公司收入增长只有在能转化为利润和股东现金流时，才支持股权价值。加密网络使用增长也必须转化为某种 token 需求、费用分配或稀缺性，才能支持 token 估值。

这一点在多层生态中尤其重要。用户可能在某个应用中交易，费用被应用拿走；应用运行在某个 L2，排序器捕获部分收入；L2 再向 L1 支付数据成本；L1 token 通过燃烧或质押获得部分价值。价值沿着技术栈分配，不能因为“生态很活跃”就自动归给某一个 token。

## 链上数据的优点和局限

链上数据的优点是透明、实时、可验证。传统金融中，很多交易和资产负债表数据滞后披露；链上数据可以即时观察。研究者和投资者可以构建新因子，如费用增长、地址活跃、资金流入、质押变化和大户行为。

局限是链上数据不包含全部信息。团队治理、法律风险、链下收入、交易所负债、真实用户身份、做市协议和监管风险都可能在链下。链上透明不能替代完整尽调。

## 小结

链上基本面指标为加密资产定价提供新数据，包括活跃地址、交易量、TVL、手续费、稳定币供应、质押率和流动性深度。网络效应会放大用户增长，也会在信心逆转时反向收缩。分析链上数据必须看生成机制，区分真实需求和补贴刷量。更重要的是，网络使用必须通过费用、燃烧、质押、治理或现金流转化为 token 价值，否则基本面增长未必属于 token 持有人。
