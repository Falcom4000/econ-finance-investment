# 40.3 稳健损失函数与多资产风险评估

来源：

- 主线：Paleologo《The Elements of Quantitative Investing》Ch.5
- 相关旧笔记：本笔记 Ch.30, Ch.37, Ch.39

风险模型评估真正困难的地方，不是写出某个误差指标，而是让这个指标和投资任务一致。一个模型可能在单资产波动率上表现好，却在组合优化中表现差；一个模型可能平均误差小，却在某些组合方向上严重低估风险。多资产风险评估必须从“模型准不准”进一步问“对我们的组合管理准不准”。

本节把稳健损失函数和多资产评估方法放在一起。稳健损失函数解决“如何比较预测和实现风险”；多资产评估解决“在哪些组合上比较”。

## 稳健损失函数要解决什么问题

假设真实方差是 sigma^2，模型预测为 sigma_hat^2。但真实方差不可见，我们只能用 realized proxy，例如收益平方 r^2 或实现方差。若 realized proxy 是无偏的，直觉上可以拿它替代真实方差评价模型。

问题是，不是所有损失函数在这种替代下都保持模型排序。一个损失函数如果不稳健，可能出现这种情况：模型 A 对真实方差更准确，但用 realized proxy 评价时，模型 B 看起来更好。这会让研究者选择错误模型。

稳健损失函数的意义就在于：只要代理变量无偏，用代理变量比较模型，排序就和用真实方差比较一致。这对金融风险评估很实用，因为我们永远只能用 noisy proxy 做事后检验。

## QLIKE 的不对称惩罚

QLIKE 常用于波动率预测评估。它和正态分布的负对数似然有关。如果模型预测方差太小，而实际收益很大，损失会很高；如果模型预测方差太大而实际收益很小，也会有损失，但形态不同。

这种不对称性符合风险管理直觉。低估风险通常比高估风险更危险。高估风险会让组合过于保守，损失机会；低估风险可能导致仓位过大、杠杆过高、触发止损或保证金问题。

当然，过度高估风险也有成本。若模型长期高估波动率，策略仓位会过小，资本效率下降。因此，QLIKE 不是只惩罚低估，而是在似然意义下衡量预测方差与实际收益的匹配。

## MSE 的直观性

MSE 更容易解释：预测方差和 realized proxy 的差距越大，损失越大。它对大误差惩罚更重，适合衡量平均预测偏差。

MSE 的优点是透明；缺点是对极端 realized proxy 可能敏感。金融收益厚尾，少数大波动日可能主导 MSE。此时研究者要判断，这种主导是合理的风险信号，还是某些异常数据或一次性事件导致的样本不稳定。

实践中，QLIKE 和 MSE 常一起看。如果两个指标都支持某模型，结论更稳。如果指标分歧，就要回到用途：是更怕低估风险，还是更重视平均误差？是用于风险限额，还是用于组合优化？

## 真实生产策略测试

最贴近应用的多资产测试，是把风险模型放进真实投资流程。用模型 A 生成组合，用模型 B 生成组合，然后比较它们的实现波动率、Sharpe、PNL、回撤、交易成本和风险违规次数。

这个测试的优点是直接回答业务问题：哪个模型让策略更好？缺点是它依赖完整流程。若 alpha、优化器、成本模型和约束不变，风险模型差异才能较干净地体现；若同时改变多个模块，就无法判断结果来自哪里。

还有一个重要原则：组合必须由同一个风险模型生成，再用实际结果检验。不能用模型 A 生成组合，却拿模型 B 的风险预测评估它。那样会把生成和评价混在一起，测试不对称。

## 平均组合测试和组合基

如果没有生产策略，可以构造一组测试组合。最简单方法是随机生成组合权重，在不同时间和收益实现上计算损失。这样可以覆盖大量方向，观察模型平均表现。

但随机组合未必代表真实策略。真实组合受 alpha、约束、交易成本、行业限制和风险预算影响，不会像数学上随机向量那样分布。因此，平均组合测试结果依赖组合分布假设。

另一种做法是选择组合基，例如 eigenportfolios、行业组合、风格组合或因子模拟组合。这样计算更可控，也更有解释性。但结果仍依赖组合基选择。选了行业组合，模型在行业风险上表现会被重点考察；选了统计主成分组合，则更偏向协方差结构方向。

## Alpha 相关组合测试

风险模型最终服务于有 alpha 的组合。如果某些组合永远不会被策略持有，它们的风险预测错误也许不那么重要。于是，可以用 alpha 相关组合测试风险模型。

一种思路是用未来一段已实现收益构造“事后 alpha”组合，再看不同风险模型对这些相关组合的风险预测。这当然不能用于真实交易，因为它使用未来信息；但用于风险模型测试时，它能让测试集中在对投资有意义的组合方向上。

这种方法的优点是贴近“如果我们有收益预测，风险模型是否帮助构造更好组合”。缺点是它依赖测试设计和未来收益窗口，可能对样本敏感。因此，它应作为补充，而不是唯一标准。

## 分布似然测试

还有一种不依赖具体组合的测试，是看整个资产收益向量在模型协方差下的似然。如果假设收益在给定协方差矩阵下近似多元正态，那么预测协方差越合理，实际收益向量出现的似然越高。

这种方法的优点是避免选择测试组合；缺点是依赖分布假设。真实收益厚尾、非正态、存在跳跃和状态转换，多元正态似然只是近似。它适合作为一种全局诊断，而不应替代组合相关测试。

## 小结

稳健损失函数的目标，是在使用 noisy realized proxy 时仍能合理比较波动率预测。QLIKE 和 MSE 是常用选择，前者与似然相关并对风险错配敏感，后者直观衡量预测方差误差。实际评估中应结合使用，而不是迷信单一指标。

多资产风险评估必须选择测试组合。真实生产策略测试最贴近应用；随机组合和组合基测试覆盖范围更广但依赖假设；alpha 相关组合测试更贴近投资任务；分布似然测试提供全局诊断但依赖正态近似。好的风险模型评估通常是多指标、多场景的证据组合。
