# 63.5 加密组合的风险预算和动态仓位管理

来源：主线参考 Di Maggio《Blockchain, Crypto and DeFi》Ch.12；补充参考本笔记 Ch.29-Ch.35、Ch.38、Ch.44-Ch.49、Ch.56、Ch.62。

如果只看牛市，加密组合管理似乎很简单：多买涨得快的资产，少拿现金，尽量提高风险暴露。但真正的组合管理不是为了在顺风时显得激进，而是为了在逆风时不被迫出局。加密资产波动大，相关性在压力时期上升，杠杆和流动性风险突出，因此风险预算必须放在收益目标之前。

风险预算回答的是：这个组合最多能承受多大亏损？波动率目标是多少？单个资产上限是多少？可以使用多少杠杆？需要保留多少稳定币或现金？单一交易所、单一协议、单一稳定币风险最多占多少？

没有这些答案，组合很容易在牛市中不断加仓，在熊市中被迫卖出。

## 风险预算不是保守，而是生存条件

一个组合一年赚 100%，听起来很好；如果最大回撤 90%，很多投资者根本坚持不到反弹。另一个组合收益较低，但回撤可控、流动性充足、没有强平风险，可能更适合长期资金。

第 35 章讲绩效评价时强调，收益必须风险调整。加密组合尤其需要这样做，因为资产价格可以在很短时间内大幅波动。只看收益，会鼓励投资者不断提高 beta、杠杆和低流动性暴露。

风险预算还要和负债匹配。个人投资者可能需要现金支出，基金可能面对赎回，DAO 国库要支付开发者工资和运营费用。如果负债是短期的，资产却全部放在高波动 token 和锁仓收益策略里，就会出现期限错配。第 23 章金融危机中的流动性问题，在组合层面同样适用。

因此，现金和稳定币缓冲不是浪费。它们在牛市中拖累收益，但在压力时期提供选择权：补保证金、应对赎回、低价买入，或只是避免被迫卖出。

## 动态仓位管理不是每天猜涨跌

动态仓位管理不是要求投资者预测每一天价格，而是根据风险环境调整暴露。波动率上升、资金费率极端、稳定币脱锚、交易所风险上升、链上清算增加，都可能提示市场风险正在恶化。

例如，波动率目标策略会在波动率上升时降低仓位，在波动率下降时提高仓位。这样做不是因为它知道明天会跌，而是因为同样仓位在高波动环境中风险更大。趋势策略跌破长期均线时降低现货暴露，也是一种风险控制。

套利策略也需要动态规模。资金费率高时可以增加基差策略，费率下降或交易拥挤时减少规模。做市策略在市场深度下降、波动率上升时也应降低库存风险。

动态管理的核心是避免在风险显著上升时仍保持原有杠杆。它服务于生存，而不是短期预测。

## 分散化不能只数 token 数量

持有 30 个 token，看起来很分散；如果它们都依赖 BTC/ETH beta、美元流动性、同一交易所流动性、同一稳定币和同一批风险偏好投资者，那么压力时期可能一起下跌。

真正的分散化要看风险来源。BTC、ETH、L1 token、DeFi token、稳定币收益、资金费率策略、做市策略、RWA 收益，风险来源不同，但在危机中仍可能相关上升。平时相关性低，不代表压力时期相关性低。

这就是尾部相关性。牛市中，每个资产都有自己的叙事；熊市中，大家可能都要换成美元、补保证金或撤出交易所。组合优化如果只用平静时期的历史相关性，会高估分散效果。

第 39-41 章因子模型的思想可以迁移到这里：先识别共同风险因子，再决定是否愿意承担。资产数量只是表面，风险来源才是核心。

## 杠杆必须在组合层面统一计算

加密组合常常在多个地方同时使用杠杆。投资者可能持有现货，又在借贷协议抵押借款买入更多资产，还在永续市场开多头，同时把 LP token 质押到收益策略中。每个账户单独看似安全，组合层面可能已经高度杠杆。

例如，账户显示有 100 万美元现货资产，但其中一部分被抵押借出稳定币，稳定币又买入更多 ETH，另一个账户还开了 ETH 永续多头。若 ETH 下跌，现货亏损、抵押品价值下降、借贷健康因子恶化、永续保证金也下降。多个风险同时触发。

所以，组合管理不能只看现货余额，要统一计算净敞口、总敞口、保证金需求、借款、抵押品和清算线。否则，投资者会低估自己在同一个方向上的集中暴露。

## 流动性预算和压力测试

加密组合需要流动性预算。哪些资产可以立刻卖出？哪些在 AMM 池里卖出会有巨大滑点？哪些锁在质押、借贷或收益策略中？哪些放在可能暂停提现的交易所？哪些需要跨链桥退出？

压力测试至少应包括几类场景：BTC/ETH 单日大跌，稳定币脱锚，主要交易所暂停提现，某条链拥堵，借贷协议清算激增，资金费率从正转负，LP 池子流动性消失，监管冲击导致整体风险偏好下降。

每个情景都要问：组合净值跌多少？哪些头寸被强平？需要多少现金补保证金？哪些资产无法退出？哪些收益策略会暂停？相关性是否上升？如果用户赎回或 DAO 需要支付工资，钱从哪里来？

传统组合管理常有流动性瀑布：先用现金，再卖高流动资产，最后处理低流动资产。加密组合也需要类似顺序。若所有资产都追求高 APY，被锁在复杂策略里，组合在危机中可能没有第一层流动性。

## 操作风险也是组合风险

加密组合还有传统组合较少遇到的操作风险。多签能否及时签名？硬件钱包是否可用？RPC 是否稳定？链上授权是否安全？交易所 API 是否故障？桥是否延迟？管理员权限是否清楚？

这些不是技术团队的边缘问题，而是组合能否执行风险计划的前提。一个组合理论上有对冲方案，但危机时签名人不在线、链拥堵、交易失败，方案就只是纸面计划。

因此，风险预算应包括操作流程：谁有权限，什么情况下减仓，哪些交易需要人工批准，紧急联系人是谁，稳定币和现金分布在哪里。课本式组合管理最终要落到可执行流程。

## 小结

加密组合管理必须先定义风险预算，再谈收益目标。风险预算包括最大回撤、波动率、单资产上限、杠杆上限、稳定币缓冲、协议和交易所集中度。动态仓位管理不是短期猜涨跌，而是在风险环境变化时调整暴露。分散化不能只看 token 数量，要看共同风险因子和压力相关性。杠杆、流动性和操作风险必须在组合层面统一管理。活下来，是加密组合长期收益的前提。
