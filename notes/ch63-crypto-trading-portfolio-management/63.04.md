# 63.4 回测偏差、交易成本和策略容量

来源：主线参考 Di Maggio《Blockchain, Crypto and DeFi》Ch.12；补充参考本笔记 Ch.35、Ch.38、Ch.43-Ch.46、Ch.56、Ch.60-Ch.62。

加密策略很容易回测出漂亮曲线。原因不难理解：资产波动大，历史中有多轮暴涨，交易数据频率高，链上指标多，策略想法也多。只要不断尝试参数，总能找到一条过去表现惊人的曲线。

但投资者真正关心的不是过去曲线是否漂亮，而是未来能否交易、能否扩容、能否在压力时期活下来。回测是研究工具，不是收益保证。第 43-46 章讲量化投资时反复强调，回测最容易骗人，尤其在数据多、样本短、策略选择多的市场中。

## 加密市场历史短，制度变化快

许多加密资产的历史只有几年，甚至几个月。市场结构也变化很快：稳定币规模扩大，永续合约机制成熟，DeFi 协议出现，Layer 2 降低成本，ETF 引入传统资金，监管规则变化，中心化交易所杠杆限制调整。

这意味着，早期市场中的价差、资金费率和流动性状况，未必代表今天。一个策略在 2020-2021 年牛市中表现很好，可能只是因为当时多头杠杆需求极强、资金费率很高、竞争资金较少。等更多套利者进入，收益就会下降。

回测不能只看全样本收益，还要分市场阶段：牛市、熊市、横盘、去杠杆、稳定币危机、交易所崩盘、链上拥堵、监管冲击。若策略只在一种状态有效，就不能把它说成普遍有效。

这和传统市场演化相同。早期低效率市场可能给简单策略留下空间；随着做市商、量化资金和监管进入，收益会被压缩。加密市场只是把这个演化过程加速了。

## 幸存者偏差会让历史看起来更美

如果今天只回测仍然活跃、仍然上市、仍有流动性的 token，结果会偏乐观。许多失败 token 已经归零、退市、失去流动性或被交易所下架。如果它们从样本中消失，回测就只看到了幸存者。

这和股票市场中的幸存者偏差一样。只用今天的指数成分股回测过去，会高估策略表现，因为失败公司已经被剔除。加密市场中这个问题更严重，因为 token 死亡速度更快，项目失败和流动性枯竭更常见。

严谨回测应尽量保留当时可交易的完整资产池，包括后来失败、退市和流动性消失的资产。否则，一个“选币策略”可能只是事后站在幸存者名单里挑赢家。

## 数据窥探会制造虚假 alpha

数据窥探指研究者尝试大量指标、窗口、参数和资产池，最后挑出历史表现最好的组合。若不记录失败实验，最终策略看起来显著，实际可能只是偶然。

加密市场特别容易数据窥探。你可以测试 7 日、14 日、30 日、90 日动量；可以测试活跃地址、TVL、费用、资金费率、交易所流入；可以换不同交易所、不同资产池、不同再平衡频率。组合太多，总会有一些历史表现很好。

解决办法不是不研究，而是建立研究纪律：先有经济假设，记录实验路径，区分训练样本和测试样本，使用样本外检验，控制多重检验。策略应当先能解释为什么赚钱，再看历史数据是否支持。

## 交易成本经常吃掉纸面收益

加密交易成本不只是交易所手续费。还包括买卖价差、AMM 滑点、Gas、桥费、借币费、资金费率、提现费、失败交易成本、MEV 和平台资金占用成本。

短频策略尤其容易被成本吞掉。回测用中间价成交，看起来每次赚 0.3%；实盘中买卖价差 0.2%、手续费 0.1%、滑点 0.2%、Gas 和失败交易再加一点，净收益可能变成负数。

成本还会随市场状态变化。平静时期价差小、Gas 低、池子深；危机时期价差扩大、Gas 上升、池子失衡、桥和交易所拥堵。若回测用平均成本，就会低估最需要交易时的真实成本。

投资学中常说，纸面 alpha 不等于可实现 alpha。加密市场里，这句话尤其重要。链上机会可以公开看到，但公开看到不等于能低成本执行。

## 策略容量决定能管理多少钱

容量指一个策略可以管理多少资金而不显著损害收益。小资金可以在低流动性 token 中进出，大资金一买就推高价格，一卖就压低价格。规模扩大后，交易意图更容易暴露，也更容易被 MEV 或其他交易者针对。

资金费率和基差策略也有容量。进入资金越多，永续空头增加，资金费率下降；跨市场套利者越多，价差越小；做市资金越多，价差收益被竞争压低。策略收益和策略规模不是独立的。

这对资管很关键。一个 100 万美元容量、年化很高的策略，未必能支撑大型基金；一个收益较低但容量大、回撤可控的策略，可能更适合机构。不能只问历史 Sharpe 多少，还要问这个 Sharpe 能在多大规模下实现。

容量估计需要看成交量、订单簿深度、AMM 池子深度、借贷额度、资金费率弹性和冲击成本。凭感觉说“还能放大十倍”是危险的。

## 回测要接近生产环境

实盘交易会遇到 API 故障、交易所维护、钱包签名、多签延迟、链拥堵、合约升级、RPC 失败、风控误触发。回测不能覆盖所有操作风险，但至少不能建立在明显不可执行的假设上。

如果实盘需要用限价单，回测就不能假设所有中间价成交；如果实盘需要跨链桥，回测就要考虑桥延迟和失败；如果实盘资金在中心化交易所，回测就要考虑提现限制和平台风险；如果实盘用链上交易，回测就要考虑 Gas、滑点和 MEV。

生产一致性不是为了让回测更难看，而是为了让回测更接近真实。一个在真实约束下仍然有效的策略，才有投资价值。

## 小结

加密策略回测面临历史短、制度变化快、幸存者偏差、数据窥探、交易成本高、容量有限和操作风险等问题。漂亮历史曲线不能证明策略可行。严谨回测需要完整历史资产池、point-in-time 数据、研究记录、样本外检验、阶段分析、真实交易成本模型、容量估计和接近生产环境的执行假设。策略只有在扣除成本、考虑规模和压力状态后仍有收益，才可能转化为真实投资能力。
