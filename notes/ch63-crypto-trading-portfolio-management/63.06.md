# 63.6 绩效归因：Beta、流动性、杠杆和 Alpha

来源：主线参考 Di Maggio《Blockchain, Crypto and DeFi》Ch.12；补充参考本笔记 Ch.35、Ch.38-Ch.41、Ch.43-Ch.49、Ch.56、Ch.62。

一个加密策略赚钱后，最容易出现的误解是把所有收益都归功于“能力”。牛市中，许多组合都会赚钱；高 beta 资产涨得更多；小市值 token 涨得更猛；杠杆会进一步放大收益；空投和流动性挖矿可能贡献大量 PnL。若不拆开这些来源，就无法判断策略到底有没有真正 alpha。

绩效归因要回答的不是“赚了多少”，而是“为什么赚”。这个问题比收益率本身更重要，因为只有知道收益来源，才知道它未来是否可能重复。

## 先问是不是市场 Beta

Beta 是对系统性风险的暴露。在传统股票市场中，一个组合如果只是持有高 beta 股票，牛市中涨得多并不奇怪；熊市中也会跌得多。加密市场也一样。BTC 和 ETH 往往是核心市场风险因子，许多 token 的涨跌可以很大程度上由它们解释。

若一个基金在 BTC 上涨 100% 的年份赚了 80%，表面收益很高，实际可能跑输基准。若另一个策略在市场下跌中只跌 5%，并保持流动性和低杠杆，反而可能体现风险控制能力。

这和 CAPM 及多因子模型的思想一致。先解释共同市场风险，再看剩余收益。不能把承担市场风险获得的回报，直接称为 alpha。

## 风格因子可能伪装成选币能力

许多加密策略偏好小市值、低流动性、高叙事弹性的 token。牛市中，这些资产可能涨得非常快。策略看起来像选币能力很强，但收益可能来自小市值风险、流动性风险和高 beta。

流动性差的资产在压力中难以卖出，买卖价差大，滑点高，容易被操纵。投资者要求更高预期收益，才愿意持有这类资产。如果策略收益来自承担这种风险，就应归因为流动性溢价，而不是全部归为 alpha。

传统市场中，小盘股和低流动性资产也可能有长期溢价。关键是，这种溢价是风险补偿，不是免费午餐。加密市场中，自由流通量低、做市集中、交易深度依赖少数场所，使这种风险更突出。

绩效归因时，要问：如果扣除小市值、动量、流动性和波动率暴露，策略还剩多少收益？如果真实卖出会造成很大滑点，账面收益是否还能实现？

## 杠杆会把 beta 伪装成 alpha

杠杆能让普通市场暴露看起来像优秀策略。一个策略用 3 倍杠杆持有 ETH，牛市中可能远超现货指数。但这不是 alpha，而是放大 beta。

评价杠杆策略，要把未杠杆收益、融资成本、保证金压力和强平风险拆开。若去掉杠杆后 Sharpe 并不高，收益主要来自承担更大风险。若杠杆引入尾部亏损，普通波动率还可能低估风险。

第 38 章讲过，高收益不等于好绩效。加密策略尤其容易在牛市中把杠杆收益误认为能力。真正的能力应体现在风险调整后收益、回撤控制、成本后表现和样本外稳定性上。

## 补贴和空投要单独归因

DeFi 策略经常从 token 激励、空投、流动性挖矿和治理奖励中获得收益。这些收益可能很可观，但它们和持续 alpha 不同。

若收益来自一次性空投，不能简单年化；若收益来自治理 token 排放，要看排放能持续多久，卖出是否有流动性，token 价格是否会被排放压低。若策略承担了智能合约、锁仓、稳定币和流动性风险，补贴只是风险补偿的一部分。

传统基金也会区分经常性收益和一次性收益。加密策略更需要这样做，因为很多历史 PnL 来自早期激励，而激励本身会随着参与者增加而下降。

## 流动性服务收益不是无风险手续费

LP、做市、基差和资金费率策略常被包装成市场中性收益。它们确实可能降低方向暴露，但不等于无风险。

LP 收益来自手续费和激励，但承担无常损失、池子失衡和合约风险；做市赚买卖价差，但承担库存风险和不利选择；基差策略赚资金费率，但承担保证金、平台和费率变化风险；保险基金收益来自承保尾部风险。

这些收益应归为流动性服务或风险承保，而不是简单归为 alpha。若策略提供了市场需要的服务，收益有经济基础；但服务背后承担的风险必须被识别。

## 一个实用归因表

可以把加密策略收益拆成几类：

| 来源 | 常见表现 | 需要追问 |
| --- | --- | --- |
| 市场 beta | BTC/ETH 上涨带动组合上涨 | 是否只是市场方向暴露 |
| 风格因子 | 小市值、动量、低流动性资产上涨 | 是否承担可解释风险溢价 |
| 杠杆 | 收益和亏损被放大 | 去杠杆后表现如何 |
| 流动性服务 | LP、做市、基差、资金费率 | 手续费或费率是否补偿风险 |
| 补贴 | 空投、挖矿、治理奖励 | 是否可重复、能否兑现 |
| Alpha | 选币、时机、执行、信息优势 | 控制以上因素后是否仍存在 |

这张表不是为了机械分类，而是为了防止把所有赚钱都说成能力。投资者真正关心的，是控制可解释风险后还剩什么。

## 绩效评价还要看路径

两个策略最终都赚 30%，路径可能完全不同。一个过程平稳、回撤小、资金可随时退出；另一个先跌 70%，最后靠一次反弹回本。对有赎回压力或现金需求的投资者来说，这两个策略不是同一种风险。

因此，绩效评价要看成本后收益、最大回撤、尾部损失、Sharpe、信息比率、换手率、交易成本、容量、流动性和样本外表现。收益路径影响投资者能否坚持，也影响基金是否会被迫在低点减仓。

最终，归因的目的不是证明策略过去赚钱，而是判断未来可重复性。来自市场 beta 的收益需要继续承担市场风险；来自补贴的收益可能消失；来自流动性溢价的收益受容量限制；来自真实信息和执行优势的 alpha 才更可能持续。

## 小结

加密绩效归因要把收益拆成市场 beta、风格因子、流动性溢价、杠杆、补贴和真正 alpha。BTC/ETH beta、小市值和低流动性暴露、永续杠杆、空投和流动性挖矿都可能解释历史收益。只有在控制风险、成本、容量、杠杆和补贴后仍然存在的超额收益，才更接近 alpha。绩效评价的目标不是只看赚了多少，而是理解为什么赚钱、能否重复、坏状态下会怎样。
