# 63.6 绩效归因：Beta、流动性、杠杆和 Alpha

来源：主线参考 Di Maggio《Blockchain, Crypto and DeFi》Ch.12；补充参考本笔记 Ch.35、Ch.38-Ch.41、Ch.43-Ch.49、Ch.56、Ch.62。

一个加密策略赚钱后，最重要的问题不是“赚了多少”，而是“为什么赚”。收益可能来自市场整体上涨、承担高 beta、持有小市值高波动资产、赚取流动性溢价、使用杠杆、拿到 token 补贴，或者真的有 alpha。绩效归因就是把这些来源拆开。

## 先区分 Beta 和 Alpha

Beta 是对系统性风险的暴露。一个组合如果主要持有 BTC、ETH 和高 beta 山寨币，在牛市中上涨很正常。这不一定是经理能力，而可能只是承担市场风险。

Alpha 是在控制系统性风险、交易成本和约束后仍然存在的超额收益。若一个策略声称有 alpha，就要证明它不是简单市场 beta、动量、规模、流动性或杠杆暴露。

加密市场中，BTC 和 ETH 往往是核心风险因子。许多 token 的收益可以很大程度上由 BTC/ETH beta 和市场状态解释。归因的第一步，是看组合对这些核心因子的暴露。

## 流动性和小市值溢价

很多加密策略偏好小市值、低流动性、高叙事弹性的 token。这类资产牛市中可能上涨很快，回测表现很好。但收益可能是流动性风险和小市值风险的补偿，而不是可持续 alpha。

流动性差的资产在压力中难以卖出，买卖价差和滑点高，容易被价格操纵。投资者要求更高预期收益，才愿意持有。若策略收益来自承担这种风险，就应归因为流动性溢价。

这和传统小盘、流动性因子类似。不能把承担难以交易资产的补偿，全部称为选币能力。

## 杠杆归因

杠杆会放大收益。一个策略若用 3 倍杠杆持有市场 beta，牛市中可能远超现货指数；但这不是 alpha，而是放大了 beta。绩效评价必须把杠杆调整回来。

可以比较未杠杆收益、杠杆后收益、融资成本和强平风险。若去掉杠杆后策略 Sharpe 并不高，收益主要来自承担更大风险。若杠杆增加尾部亏损，普通波动率还可能低估风险。

第 38 章讲过，高收益不等于好绩效，必须看每单位风险收益。加密策略尤其要防止把杠杆牛市收益误认为能力。

## 补贴和空投归因

DeFi 策略可能从 token 激励、空投、流动性挖矿和治理奖励中获得收益。这些收益应单独归因。它们可能是真实策略能力，也可能是早期补贴红利。

若收益主要来自一次性空投，不能外推为长期年化收益；若收益来自治理 token 排放，要看排放能持续多久，卖出是否有流动性，是否伴随底层风险。补贴归因能防止把不可重复收入当作稳定 alpha。

传统基金也会区分经常性收益和一次性收益。加密策略更需要这种区分，因为激励和空投常常显著影响历史 PnL。

## 归因框架

一个简化加密绩效归因可以包括：

| 来源 | 例子 | 归因问题 |
| --- | --- | --- |
| 市场 beta | BTC/ETH 上涨 | 是否只是市场方向暴露 |
| 风格因子 | 动量、小市值、流动性 | 是否承担可解释风险溢价 |
| 杠杆 | 永续、借贷循环 | 收益是否来自放大风险 |
| 流动性服务 | LP、做市、基差 | 手续费是否补偿风险 |
| 补贴 | 空投、挖矿奖励 | 是否可重复、能否兑现 |
| Alpha | 选币、时机、执行 | 控制以上因素后是否仍存在 |

这张表的目的不是把所有收益机械归类，而是提醒投资者：策略收益必须拆开，才能判断未来是否可信。

## 绩效评价要看成本后和风险后

最终评价应使用成本后收益、风险调整指标、回撤、尾部损失、容量、流动性和样本外表现。Sharpe、信息比率、最大回撤、因子暴露、换手和交易成本都要一起看。

一个策略如果在牛市中高收益、高回撤、高杠杆、低流动性，适合的投资者很有限。一个收益较低但风险受控、归因清楚、容量稳定的策略，可能更有长期价值。

## 小结

加密绩效归因要把收益拆成市场 beta、风格因子、流动性溢价、杠杆、补贴和真正 alpha。BTC/ETH beta、小市值和低流动性暴露、永续杠杆、空投和流动性挖矿都可能解释历史收益。只有在控制风险、成本、容量和补贴后仍然存在的超额收益，才更接近 alpha。绩效评价的目标不是证明策略赚钱，而是理解它为什么赚钱、未来是否能重复。
