# 57.5 空投、流动性挖矿和增长激励的可持续性

来源：主线参考 Di Maggio《Blockchain, Crypto and DeFi》Ch.3、Ch.6、Ch.9-Ch.10；补充参考本笔记 Ch.2、Ch.5、Ch.24、Ch.31-Ch.33、Ch.53-Ch.56。

新协议最大的困难不是写出合约，而是让用户、流动性、开发者和注意力聚集起来。传统平台会补贴司机和乘客，电商会发优惠券，银行会用高利率吸引存款，券商会降低佣金获取客户。加密协议常用空投、流动性挖矿和生态激励完成类似任务。它们不是慈善，而是用 token 购买增长。

## 空投：把潜在用户变成利益相关者

空投是把 token 分发给早期用户、贡献者、测试者或目标社区。它有三个常见目的。第一，奖励早期使用者，形成公平启动叙事。第二，把用户变成治理参与者和宣传者。第三，快速扩大 token 分布，降低项目被少数人控制的印象。

空投也有明显问题。很多用户为了获得未来空投而进行“刷量”：创建多个地址、反复小额交互、桥接资产、制造虚假交易。协议看到的活跃地址和交易量，可能并不代表真实需求，而是空投预期驱动的行为。第 56 章讨论链上基本面指标时已经看到，链上数据必须理解生成机制。

空投的投资含义也复杂。若空投给真实用户，并且后续产品有留存，空投可以把用户变成社区成员；若空投只吸引短期套利者，领取后抛售会带来价格压力。空投不是免费增长，它把未来一部分所有权支付给早期地址。

## 流动性挖矿：用 token 租用资产负债表

流动性挖矿是向提供资金、交易深度或借贷流动性的参与者发放 token。DEX 给 LP 奖励，借贷协议给存款者和借款者奖励，稳定币协议给某些池子奖励，目的都是让协议看起来更深、更便宜、更容易使用。

从金融学角度看，流动性挖矿是在租用外部资产负债表。LP 把资产放入池子，承担无常损失、价格风险和智能合约风险；资金供应者把资产借给协议，承担坏账和流动性风险。协议用 token 支付他们的风险补偿。

关键问题是补贴结束后流动性是否留下。若 LP 留下是因为真实交易费足以补偿风险，协议形成可持续市场；若 LP 只是为了奖励，奖励下降后资金迅速迁移，TVL 就是租来的规模。传统银行用高存款利率也能快速扩张负债，但如果贷款收益无法覆盖资金成本，扩张越快亏损越大。

## 增长激励的会计：奖励是成本，不是收入

许多加密项目在展示数据时，会强调 TVL、交易量、用户数和年化收益率，却淡化 token 激励成本。对投资者来说，必须把激励当作费用。一个协议发放 1 亿美元 token 换来 5000 万美元真实手续费，并不说明商业模式优秀；它可能是在亏本买收入。

传统企业也会补贴增长。网约车、外卖、电商和金融科技公司都可能用补贴获取用户。区别在于，股票投资者会关注获客成本、留存率、单位经济模型和补贴退出后的利润率。DeFi 协议也需要同样分析：每单位 token 激励带来多少真实费用？用户留存多长？补贴停止后交易量下降多少？被吸引的是长期用户还是雇佣资本？

可以用一个简化关系理解：

```text
可持续增长 = 真实使用需求 + 正的单位经济模型 + 可承受的激励成本
不可持续增长 = 补贴驱动流量 + 低留存 + 高 token 稀释
```

## 激励会改变用户行为，也会扭曲指标

激励不是中性的。协议奖励什么，用户就优化什么。如果奖励 TVL，用户会存入资产但未必交易；如果奖励交易量，用户可能刷量；如果奖励借款，用户可能循环借贷；如果奖励治理参与，用户可能为了领取奖励投票而不理解提案。

经济学把这称为激励相容问题。设计者希望用户行为促进协议长期价值，但用户会根据个人收益最大化行动。若奖励指标和真实目标不一致，结果就会偏离。学校若只奖励考试分数，学生可能死记硬背；银行若只奖励贷款增长，信贷质量可能下降；协议若只奖励 TVL，资本可能短期堆积却不创造收入。

这也是为什么链上指标需要和收入、留存、风险一起看。TVL 高但费用低，可能说明资本效率差；交易量高但 LP 亏损，流动性不可持续；用户数高但空投后消失，真实需求不足。

## 生态基金和开发者激励

除了用户和 LP，协议还会补贴开发者、项目方、钱包、前端和基础设施。智能合约平台尤其依赖开发者生态。没有应用，底层链很难产生持续需求；没有工具和钱包，用户很难进入生态。

开发者激励更像长期资本开支，而不是短期营销。它的回报不一定立刻体现在费用上，却可能形成应用网络和资产沉淀。判断这类激励，要看资金是否支持了真实产品、是否形成用户留存、是否减少生态对单一团队的依赖。

问题在于，生态基金容易变成低效拨款。若治理监督弱，资金可能流向关系项目、重复建设或短期叙事。传统风险投资承担失败率，但有尽调、董事会和后续融资纪律；DAO 生态基金也需要类似的筛选、里程碑和问责机制。

## 何时激励是合理的

激励并非都不好。新网络早期存在冷启动问题：没有用户就没有流动性，没有流动性就没有用户。适度补贴可以跨过启动门槛，形成网络效应。关键是补贴是否买到了未来无需补贴也能存在的关系。

合理激励通常有几个特征：目标清楚，奖励指标接近真实价值；期限有限，不能无限依赖排放；对象精确，避免大规模雇佣资本套利；有数据评估，能比较激励前后留存和收入；与风险承担匹配，补偿的是真实服务而不是虚假活动。

## 小结

空投、流动性挖矿和生态激励是协议用 token 购买用户、流动性、开发者和网络效应的工具。它们可以帮助协议冷启动，也可能制造刷量、短期 TVL 和高稀释。激励的核心问题不是“有没有增长”，而是增长是否来自真实需求，补贴停止后是否留存，token 排放是否买到了可持续现金流和网络效应。
