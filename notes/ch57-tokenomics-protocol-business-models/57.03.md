# 57.3 协议收入、手续费分配和价值捕获

来源：主线参考 Di Maggio《Blockchain, Crypto and DeFi》Ch.2、Ch.6、Ch.9-Ch.10；补充参考本笔记 Ch.5、Ch.24、Ch.31-Ch.33、Ch.52-Ch.56。

一个协议能否捕获价值，取决于它在链上经济中扮演什么角色，以及用户愿意为这个角色支付多少费用。传统企业卖产品、收收入、扣成本、形成利润；银行赚取净息差和手续费；交易所收撮合费和上市费；基金管理人收管理费和业绩报酬。加密协议也有收入，但收入的归属路径更复杂，因为链上系统通常把费用分给多个参与者。

## 从用户支付开始，而不是从市值开始

分析协议收入要从用户为什么付费开始。用户在链上支付费用，通常是为了获得交易执行、借贷流动性、稳定币铸造、资产跨链、预言机数据、保险保障、NFT 交易或合成资产敞口。只要用户愿意付费，说明协议解决了某种经济问题。

但用户支付的总费用不等于协议利润。以 DEX 为例，交易者支付手续费，费用可能主要给 LP，协议本身只抽取一小部分，甚至暂时不抽成。以借贷协议为例，借款人支付利息，绝大部分归资金供应者，协议可能只拿储备因子。以公链为例，用户支付 Gas，部分给验证者，部分被销毁，生态应用和 Layer 2 也会分走活动。

因此，第一步要区分三层概念：

| 概念 | 含义 | 常见误读 |
| --- | --- | --- |
| 用户总费用 | 用户为使用协议支付的全部费用 | 把它直接当成协议收入 |
| 协议收入 | 协议规则实际留下来的收入 | 忽略给 LP、验证者、前端的成本 |
| token 持有人价值 | 最终能流向 token 的收入、销毁或权利 | 认为协议有收入就必然利好 token |

## 手续费分配是一张利益相关者地图

链上协议不是单一公司，而是由用户、验证者、LP、清算人、预言机、前端、治理者、做市商和开发团队共同运行。手续费分配就是这张网络的收入分配表。

如果费用过多流向用户补贴，协议可能增长快但缺少利润；如果费用过多流向 token 持有人，LP 和做市商可能离开，交易深度变差；如果费用过多流向验证者，网络安全可能更强，但用户成本更高；如果费用太低，短期用户体验好，长期安全预算和开发资金可能不足。

这和金融中介理论相连。银行、券商和交易所之所以能收费，是因为它们提供信用筛选、流动性、支付结算、风险管理和信息处理。DeFi 协议如果只是把这些功能自动化，理论上可以降低部分中介成本；但它仍需要支付安全、流动性、数据和治理成本。手续费不是越低越好，而要足以覆盖系统运行所需的真实资源。

## 价值捕获：谁拥有瓶颈，谁更可能收费

价值捕获来自稀缺和瓶颈。如果某个协议提供的服务容易被复制，用户迁移成本低，手续费就会被竞争压低；如果协议拥有深流动性、强网络效应、可信安全记录、独特数据或合规入口，它更有能力收费。

AMM 的流动性是一个典型例子。交易者愿意到深流动性池交易，因为滑点低；LP 愿意到交易量大的池子提供资金，因为手续费多。交易者和 LP 相互吸引，形成网络效应。此时，协议可能尝试抽取一部分费用。但如果抽成太高，LP 可能迁移到其他 DEX，交易者也会跟着迁移。价值捕获受到竞争约束。

公链也类似。Ethereum 有开发者、应用、资产、钱包、稳定币和安全记录形成的生态优势，所以用户愿意支付较高 Gas。但 Layer 2 和其他链降低了替代成本。于是 ETH 的价值捕获不只取决于 Ethereum 生态是否繁荣，还取决于费用在哪一层产生、基础费是否被销毁、排序收入归谁、用户是否愿意为安全支付溢价。

## 协议收入和传统企业利润的差异

协议收入与企业利润有相似之处，但不能直接等同。企业有员工、资产负债表、法定股东、税务和会计报表；协议则可能由智能合约、DAO、基金会、贡献者和外部服务商组成。许多成本并不出现在传统利润表里，而是以 token 补贴、治理拨款、漏洞赏金、流动性激励和安全风险形式存在。

例如，一个协议显示“收入”为正，但如果它同时大量发放治理 token 吸引流动性，真实经济利润可能为负。类似地，一家公司如果用高额股权激励换增长，投资者需要把股权稀释计入成本。token 激励也应被当作成本，而不是因为不是现金支出就忽略。

协议还可能把价值留在国库，而不是直接给 token 持有人。国库资金可以用于开发、补贴、安全审计、回购或生态投资。国库像公司现金，但治理质量决定其价值。若国库被低效使用或被治理攻击，账面资产未必能转化为持有人价值。

## 费用开关：从增长模式到盈利模式

许多协议早期不抽取或少抽取协议费用，目的是吸引用户和流动性。当网络效应形成后，社区可能讨论打开“费用开关”，把部分费用导向国库或 token 持有人。这个过程类似互联网平台从补贴增长转向变现。

费用开关的难点在于时机和弹性。如果用户和 LP 对费用非常敏感，抽成会导致流动性下降和交易量流失；如果协议确实拥有独特流动性和品牌信任，适度抽成可能不影响增长。经济学上，这就是需求弹性和市场势力问题。一个协议能否收费，要看用户有没有替代选择，而不是看社区是否希望收费。

费用开关还会改变 token 的监管属性。若 token 明确分享协议利润，可能更接近证券型资产；若只是治理权和使用权，监管判断可能不同。代币经济学不是纯技术问题，它会被法律制度和市场结构约束。

## 价值捕获的基本检查清单

分析协议商业模式时，可以按下面顺序检查：

1. 用户为什么付费，支付的是哪种服务？
2. 用户总费用有多少，是否来自真实需求而不是补贴循环？
3. 费用分给谁：LP、验证者、前端、协议国库、token 持有人？
4. 协议收入扣除 token 激励、安全成本和运营拨款后是否为正？
5. token 持有人凭什么分享价值：销毁、回购、分红、质押收益还是治理期权？
6. 竞争者能否复制服务并压低费用？

这个清单把加密协议拉回金融学和投资学的基本问题：现金流从哪里来，风险由谁承担，剩余价值归谁所有，估值是否已经反映增长。

## 小结

协议收入分析必须从用户支付开始，再区分用户总费用、协议收入和 token 持有人价值。手续费分配反映协议中用户、LP、验证者、清算人、前端、国库和治理者之间的利益安排。价值捕获取决于网络效应、竞争、需求弹性、费用路径和监管约束。一个协议有交易量不等于有利润，有利润也不等于 token 持有人能够获得现金流。
