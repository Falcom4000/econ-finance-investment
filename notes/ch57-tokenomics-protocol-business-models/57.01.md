# 57.1 Token 的经济权利：使用权、治理权和现金流请求权

来源：主线参考 Di Maggio《Blockchain, Crypto and DeFi》Ch.2、Ch.6、Ch.9-Ch.10；补充参考本笔记 Ch.2、Ch.24、Ch.31、Ch.33、Ch.52-Ch.56。

很多人第一次接触加密资产时，会把 token 简单理解成“某个项目发行的币”。这个说法太粗。股票代表公司剩余收益和控制权的一部分，债券代表按合同收取本息的权利，商品代表对实物或使用价值的索取。token 也必须问同样的问题：持有人到底拥有什么？他能使用什么服务？能参与什么决策？能否分享协议现金流？如果这些问题回答不清，所谓“代币经济学”就容易变成价格叙事。

这一步要慢下来，因为很多 token 的价格故事会先于权利设计出现。市场可能先说“这是某条链的未来”“这是某个协议的治理权”“这是社区所有权”，但投资学不能停在口号上。只有当权利能够被规则、合约、治理程序或法律结构确认时，它才更接近可分析的经济权益；如果权利只是未来也许会发生的期待，价格中就包含很大的期权和叙事成分。

## 使用权：进入网络和消耗服务的凭证

最基础的一类经济权利是使用权。ETH 用来支付 Ethereum 上的 Gas，LINK 可以和预言机服务、节点激励联系起来，稳定币可以在链上支付和结算，某些游戏或内容平台 token 可以购买虚拟物品或服务。使用权的特点是：需求来自用户想完成某种链上活动，而不是单纯想持有资产。

使用权和经济学中的“派生需求”很像。企业需要劳动力，不是因为劳动力本身放在仓库里有价值，而是因为劳动能生产商品；用户需要 Gas，不是因为 Gas 这个动作本身有价值，而是因为它能执行转账、交易、借贷、铸造 NFT 或调用合约。网络使用越频繁，对原生资产或服务 token 的交易性需求越强。

但使用权不自动等于投资价值。一个平台可以非常有用，但如果每次使用只需要极少 token，并且 token 流通速度很高，用户用完即卖，那么持有需求可能有限。传统金融里也有类似例子：支付网络处理大量交易，不代表每一张预付卡本身能成为高收益资产。投资者必须区分“协议有用”和“token 捕获价值”。

## 治理权：改变规则的投票权

第二类权利是治理权。治理 token 通常允许持有人对协议参数、资金库支出、升级方案、抵押品名单、手续费分配、风险参数等事项投票。MakerDAO 的抵押率和清算参数、许多 DeFi 协议的费用开关、DAO 的资金使用，都体现了治理权的经济含义。

治理权看似像股票投票权，但差异很大。股票投票权通常嵌入公司法、董事会、信息披露和司法执行体系。治理 token 的投票则依赖智能合约、论坛讨论、委托投票、链上或链下快照、基金会和核心团队的协调。它可能更开放，但也可能更脆弱：投票率低、大户集中、信息不对称、短期投机者和长期建设者目标不一致，都会削弱治理质量。

治理权的价值来自“规则可被改变”。如果某个协议未来能够产生大量手续费，治理权可能决定这些费用是否分给 LP、质押者、国库或 token 持有人。即使当前没有现金流分配，市场也可能给治理权定价，因为持有人拥有未来改变分配规则的可能性。不过这种价值高度不确定：监管限制、社区反对、竞争压力和安全需求，都可能阻止费用最终流向 token。

## 现金流请求权：最接近证券分析的部分

第三类权利是现金流请求权。若 token 持有人能够明确收到协议收入、回购、分红、费用分成或抵押收益，那么分析就开始接近股票、债券或基金份额。现金流越清晰，估值越能使用传统金融工具，例如收入倍数、折现现金流、风险溢价和剩余收益。

但加密协议中的现金流经常存在三层拆分。第一层是用户支付的总费用，例如交易手续费、借贷利差、清算罚金、铸造费。第二层是协议真正留下来的收入，即扣除给验证者、LP、做市商、清算人、前端、推荐人和补贴之后的净收入。第三层是 token 持有人能否获得这部分收入。很多项目有使用量，却没有净收入；有净收入，却不分给 token；承诺未来分配，但治理上尚未执行。

这也是投资学中的基本面分析进入加密资产的入口。股权估值关心的是企业自由现金流能否归属于股东，债券估值关心本息能否按约支付，加密 token 估值则要问协议收入是否可持续、成本由谁承担、剩余价值是否真正流向 token 持有人。

## 同一个 token 可能同时包含多种权利

现实中的 token 往往不是单一权利。ETH 同时是 Gas 支付资产、PoS 质押资产、DeFi 抵押品和平台经济敞口；治理 token 可能同时用于投票、质押、获得费用分成和激励流动性；RWA token 可能代表某个真实资产的收益权，同时还需要链上转让和合规限制。

这种多重属性会提高分析难度。若一个 token 同时有使用需求和投资需求，价格上涨会提高持有者财富，却可能提高用户使用成本；若 token 既是治理票又是抵押品，价格下跌会削弱治理稳定性和系统偿付能力；若 token 既用于激励增长又代表稀缺权益，持续增发会帮助生态扩张，却稀释老持有人。

可以用一个简单框架整理：

| 权利类型 | 持有人获得什么 | 主要风险 | 对应的传统金融类比 |
| --- | --- | --- | --- |
| 使用权 | 支付 Gas、访问服务、参与网络 | 使用量不足、替代链竞争、流通速度太高 | 预付费服务、平台通行证 |
| 治理权 | 对协议规则和资金使用投票 | 投票冷漠、巨鲸控制、治理攻击 | 股东投票权、会员制组织 |
| 现金流请求权 | 费用分成、回购、收益分配 | 现金流不可持续、监管约束、分配规则改变 | 股票、基金份额、收益权凭证 |
| 抵押或安全权 | 通过质押维护网络并获得奖励 | 罚没、验证者集中、收益率下降 | 保证金、资本金、保险准备 |

## 为什么权利定义决定估值方式

如果 token 只有使用权，估值重点是网络使用量、费用需求、货币流通速度和替代品竞争。如果 token 主要是治理权，估值重点是未来规则控制权、协议国库、费用开关和治理集中度。如果 token 有现金流请求权，估值可以更直接地进入收入、利润、折现率和风险溢价。若 token 只是补贴工具，价值则更依赖新用户、新资金和市场情绪。

因此，代币经济学的第一步不是问“这个币会不会涨”，而是问“这个 token 把哪些经济权利写进了规则，哪些只是市场期待”。前者更接近合约、制度和现金流，后者更接近叙事和期权。两者都可能影响价格，但投资含义完全不同。

这个区分也能保护投资者不把“社区共识”误当成“现金流权利”。社区可以创造需求、传播叙事、参与治理，但社区本身不等于可分配收益。若 token 没有清晰的费用路径、使用刚性或治理控制权，价格上涨更多依赖新增买盘和预期延续，而不是已经形成的资产权利。

## 小结

Token 不是一个统一资产类别，而是一组经济权利的组合。使用权来自网络服务需求，治理权来自改变协议规则的能力，现金流请求权来自协议收入向持有人的分配。一个项目很有用，不代表 token 一定有投资价值；一个 token 价格上涨，也不代表协议已经产生真实剩余。分析代币经济学时，必须先拆清权利，再讨论供给、激励、收入和估值。
