# 61.6 链上收益和传统固定收益、货币市场收益的比较

来源：主线参考 Di Maggio《Blockchain, Crypto and DeFi》Ch.8；补充参考本笔记 Ch.20-Ch.21、Ch.24、Ch.32、Ch.35、Ch.38、Ch.57-Ch.60。

链上收益常被拿来和银行存款、货币基金、债券收益率比较。但“年化收益率”只是形式相同，背后的风险完全不同。要理解链上收益，必须把它拆成传统金融能识别的部分：无风险利率、信用风险、流动性溢价、市场风险、操作风险和补贴。

## 传统固定收益的收益来自什么

债券和货币市场工具通常承诺未来现金流。国库券收益接近短期无风险利率，公司债收益包括无风险利率加信用利差，长期债券还包含期限溢价和利率风险。货币基金收益来自持有短期债务工具，强调流动性和本金稳定，但仍有信用和流动性风险。

这些收益有法律契约支持。债券发行人必须按契约付息还本；银行存款有银行资产负债表和监管制度；货币基金有投资限制、披露和赎回规则。风险不是没有，而是有较成熟的法律和监管框架。

链上收益则更多来自智能合约规则、市场费用、token 排放、质押奖励和用户行为。它不一定有法律偿付承诺，也不一定有最终贷款人或存款保险。

## 链上收益的主要类型

链上收益可以分为几类。借贷收益来自借款人支付利息和协议奖励；LP 收益来自交易手续费、token 激励和资产价格变化；质押收益来自网络发行、交易费和验证服务；收益农场收益来自多协议组合；再质押收益来自为额外服务承担安全责任。

每一类都要问收益付款方是谁。借贷利息由借款人付，交易手续费由交易者付，质押发行由全体 token 持有人稀释承担，协议奖励由国库或新增发行承担，再质押收益由使用安全服务的协议或补贴支付。

付款方越清晰，收益越容易分析；付款方越依赖新发行和新资金，收益越需要谨慎。

## 为什么链上收益常高于传统收益

链上收益高，可能有合理原因。智能合约风险、私钥风险、稳定币风险、价格波动、清算风险、监管风险和流动性风险都需要补偿。市场参与者要求更高收益，才愿意承担这些风险。

高收益也可能来自补贴。协议为了增长发放 token，短期 APY 被推高。只要 token 价格高，显示收益就高；一旦奖励减少或 token 下跌，收益会迅速下降。

还有一种情况是杠杆和循环交易抬高收益。用户借入资产参与策略，推高借款需求和利率，同时获得奖励。表面上池子收益很高，底层却是杠杆系统互相喂养。压力时期，这类收益可能和清算风险一起消失。

## 链上收益不是无风险利率

在传统资产定价中，无风险利率是很多估值和组合分析的基准。链上稳定币借贷利率有时被称为“链上无风险利率”，但这个说法要非常谨慎。稳定币本身有发行方、储备、合规和脱锚风险；借贷协议有合约和清算风险；链上提款有流动性和拥堵风险。

更合理的说法是：某些大型稳定币借贷市场可以反映链上美元流动性的机会成本，但它不是纯无风险利率。它包含风险溢价、流动性状态和 DeFi 杠杆需求。

投资者若把链上收益当作现金替代品，可能低估尾部风险。现金管理首先关注本金安全和流动性，收益只是第二位；许多链上策略则把本金暴露在复杂协议风险中。

## 用固定收益框架分析链上收益

可以把链上收益拆成：

```text
链上收益 = 基础美元利率 + 协议风险溢价 + 流动性溢价 + 市场风险补偿 + token 补贴 + 杠杆需求溢价
```

这个公式不是精确模型，而是分析清单。基础美元利率来自传统货币市场环境；协议风险溢价补偿合约、治理和预言机风险；流动性溢价补偿退出困难；市场风险补偿来自资产价格波动；token 补贴来自协议排放；杠杆需求溢价来自交易者愿意为借款支付更高利率。

拆开之后，投资者才能判断收益是否可持续。若收益主要来自基础利率和真实借款需求，相对稳健；若主要来自 token 补贴和杠杆循环，周期性更强；若主要补偿尾部风险，必须看极端损失是否可承受。

## 组合管理中的位置

链上收益资产在组合中不能简单归为“固定收益”。稳定币借贷可能更接近高风险货币市场策略；LP 头寸更像带期权特征的做市策略；质押更像持有高波动资产并获得安全服务收入；收益农场更接近多策略对冲基金或结构化产品。

因此，配置链上收益要先定义目标。如果目标是现金管理，应选择风险最低、流动性最高、透明度最强的工具，并接受较低收益。如果目标是获取风险溢价，可以承担更多协议和市场风险，但要给出风险预算、仓位上限和压力测试。

## 小结

链上收益和传统固定收益都用年化收益率表达，但风险结构不同。传统固定收益依赖法律契约、发行人信用和监管框架；链上收益依赖智能合约、市场费用、token 激励、质押安全和用户行为。链上高收益可能来自真实风险补偿，也可能来自补贴和杠杆循环。它不能简单视为无风险利率或现金替代品，必须按收益来源、风险溢价、流动性和尾部风险拆解。
