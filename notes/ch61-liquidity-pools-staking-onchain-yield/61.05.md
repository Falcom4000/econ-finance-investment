# 61.5 收益农场、杠杆循环和可持续性问题

来源：主线参考 Di Maggio《Blockchain, Crypto and DeFi》Ch.8；补充参考本笔记 Ch.13、Ch.24、Ch.35、Ch.38、Ch.57-Ch.60。

收益农场指用户在多个 DeFi 协议之间移动资金，追逐手续费、利息、token 奖励和质押收益。这个名称听起来像把资金“种下去”等收成，但金融上更准确的说法是：用户在不同协议间配置风险资产和流动性服务，获取风险补偿和补贴。

## 收益农场怎样形成

早期 DeFi 协议为了吸引用户和流动性，会向存款者、借款者、LP 或质押者发放治理 token。用户发现，自己不仅能获得借贷利息或交易手续费，还能获得额外奖励。于是资金开始在收益最高的池子之间迁移。

一个简单收益农场路径可能是：用户存入 USDC 到借贷协议，获得存款利息和奖励 token；再借出另一种资产，去 DEX 提供流动性；拿到 LP token 后质押到农场合约，获得更多奖励。每一步都增加收益来源，也增加风险来源。

这体现了 DeFi 可组合性。金融策略不再由单一机构提供，而是由多个协议拼接。用户和收益聚合器可以像组合积木一样，把借贷、兑换、LP、质押和奖励叠在一起。

## 杠杆循环如何放大收益

收益农场常见做法是杠杆循环。用户存入资产作为抵押，借出资产，再把借出的资产换回或存回，增加总头寸。例如，存入 ETH 借 USDC，用 USDC 买更多 ETH，再存入继续借。这样可以放大对 ETH 的敞口。

若收益率高于借款成本，循环看起来很有吸引力。若 token 奖励同时补贴存款和借款，用户甚至可能因为借款而获得净奖励。这个机制会迅速推高 TVL 和利用率。

但杠杆循环放大清算风险。底层资产价格小幅下跌，就可能触发连环清算；借款利率上升，也可能使策略收益转负。第 59 章讲过，利用率上升会推高利率，收益农场本身可能改变它依赖的利率环境。

## 高 APY 的来源

高 APY 可能来自多种来源：

1. 真实用户支付的交易费或借款利息。
2. 协议发放的 token 补贴。
3. 杠杆放大后的风险暴露。
4. 流动性差或智能合约风险的风险溢价。
5. 新资金进入推高奖励 token 价格。

这些来源的质量不同。真实费用更可持续，补贴取决于国库和发行计划，杠杆收益在压力中可能反转，风险溢价要求投资者承担尾部损失，新资金驱动的收益则容易形成投机循环。

因此，看到高 APY 时，第一反应不应是“收益很好”，而应问“谁在付钱，为什么愿意付，能付多久，风险在哪里”。

## 收益聚合器：自动化策略管理

收益聚合器会替用户在不同协议之间寻找收益、复投奖励、调整仓位和优化 Gas。它类似链上基金或自动化资产管理工具。用户把资金交给金库，金库按照策略执行。

聚合器降低了用户操作成本，也引入策略风险。用户需要信任策略代码、策略开发者、治理和风险控制。若策略过度拥挤，大家同时退出会放大滑点和清算；若底层协议出问题，聚合器会把风险集中传给所有金库参与者。

传统基金监管关注经理代理问题、费用透明和风险披露。链上聚合器也有类似问题，只是表现为策略合约、治理权限、绩效费、审计和参数透明。

## 可持续性检验

收益农场是否可持续，取决于收益是否来自真实经济活动。如果交易者持续支付手续费，借款人持续支付利息，网络使用真实增长，那么 LP 和存款者有可能获得长期风险补偿。若收益主要来自新 token 排放和新用户进入，持续性就弱得多。

还要看风险是否被正确定价。一个策略平时每天稳定赚小钱，但偶尔因脱锚、清算、桥攻击或合约漏洞损失本金，表面 APY 会严重低估尾部风险。投资绩效评价不能只看平均收益，要看波动、回撤、偏度、流动性和最大损失。

这和对冲基金、结构化产品和高收益债分析类似。高收益往往意味着承担了某种不明显的风险，只有在压力状态下才暴露。

## 小结

收益农场是用户在多个 DeFi 协议间配置资金，获取利息、手续费、token 奖励和质押收益的策略。可组合性让收益来源叠加，也让风险来源叠加。杠杆循环可以放大 APY，但同时放大清算和利率风险。高收益必须拆解来源：真实费用、补贴、杠杆、流动性溢价还是新资金推动。可持续收益应来自真实经济活动和可承受风险，而不是无限 token 排放。
