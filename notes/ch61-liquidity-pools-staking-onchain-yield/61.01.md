# 61.1 流动性池如何形成可交易市场

来源：主线参考 Di Maggio《Blockchain, Crypto and DeFi》Ch.8；补充参考本笔记 Ch.24、Ch.35、Ch.38、Ch.44、Ch.57-Ch.60。

上一章讲 AMM 时，重点放在交易者视角：为什么不需要订单簿，为什么恒定乘积公式能报价，为什么会有滑点和 MEV。本章转到资金提供者视角。一个 AMM 池子之所以能让交易者随时买卖，是因为有人先把资产放进池子。这个人就是流动性提供者，通常简称 LP。

## 流动性不是自然存在的

传统市场中，流动性来自多种参与者。交易所集中订单，做市商持续报价，机构投资者提供大额交易对手，券商和清算机构降低交易摩擦。流动性看起来像市场自然属性，实际上是一种需要资本、风险管理和激励支持的服务。

AMM 把这项服务开放给任何持有资产的人。只要用户把两种资产按规则存入池子，例如 ETH 和 DAI，池子就可以为其他人提供兑换。交易者向池子支付手续费，LP 按份额分享手续费。

这和共同基金有一点相似：许多投资者把资金汇入一个集合，集合再对外提供某种金融功能。共同基金提供分散投资，AMM 流动性池提供交易深度。区别在于，AMM 池子不是替 LP 主动选股或选债，而是按合约规则被动面对交易流。

## LP token 是池子份额凭证

用户向池子存入资产后，通常会收到 LP token 或某种头寸凭证。它代表用户对池子的一部分所有权。若池子总价值增长，LP token 的价值也增长；若池子遭遇损失，LP token 也会受损。用户退出时，交回 LP token，按份额取回池中资产。

LP token 的重要性在于，它把流动性头寸变成可组合资产。LP token 可以被其他协议识别、质押、抵押或用于收益农场。这就是 DeFi “金融积木”的一部分：一个协议中的头寸，成为另一个协议的输入。

但凭证化也让风险传播更容易。若某个 LP token 被用作借贷抵押品，而底层池子价格波动或被攻击，风险会传导到借贷协议。可组合性提高资金效率，也扩大了风险链条。

## 流动性池怎样吸引交易者

交易者选择某个池子，通常看三个因素：价格、滑点和费用。价格由池子储备和套利决定，滑点由池子深度决定，费用由协议参数决定。一个深池可以让交易者用较低价格冲击完成交易，因此更有吸引力。

交易量又反过来吸引 LP。交易越多，手续费越多，LP 更愿意提供资金。LP 增加后，池子更深，滑点更低，又吸引更多交易者。这是一种网络效应。

```mermaid
flowchart LR
    A[更多LP存入资产] --> B[池子更深]
    B --> C[滑点更低]
    C --> D[更多交易者使用]
    D --> E[手续费收入增加]
    E --> A
```

这个循环解释了为什么早期 DEX 要用 token 激励吸引流动性。没有初始深度，交易者不来；交易者不来，手续费不足，LP 也不来。补贴试图帮助池子跨过冷启动门槛。

## 流动性质量不只看 TVL

TVL 是总锁仓价值，常被用来衡量 DeFi 规模。但在流动性池中，TVL 不是全部。一个池子 TVL 很高，若资产波动大、交易量低、手续费低，LP 收益未必好；一个池子 TVL 较低，但交易活跃、资产相关性高、费用稳定，可能更有质量。

流动性质量至少要看：池子深度、日交易量、交易量与 TVL 比、资产相关性、手续费等级、套利频率、价格冲击、LP 集中度和外部市场深度。投资者不能把“锁了很多钱”直接等同于“安全”或“有价值”。

这和传统基金分析类似。基金规模大不等于收益好，资产配置、费用、流动性和风险暴露更重要。AMM 池子的 TVL 只是规模指标，不是绩效指标。

## 流动性池是市场基础设施，也是风险资产

对交易者而言，流动性池是交易基础设施。对 LP 而言，它是一个风险资产组合：持有两种或多种资产，自动再平衡，收取手续费，承担价格波动、合约和无常损失风险。

这两种视角必须同时存在。若只从交易者视角看，会低估 LP 承担的风险；若只从 LP 收益看，会忘记池子的社会功能是提供市场流动性。金融市场中，做市、交易所和清算系统本来就同时具有公共基础设施和私人盈利业务的双重属性。

## 小结

流动性池通过 LP 存入资产形成可交易市场。LP token 代表池子份额，并让流动性头寸可以在其他协议中使用。池子深度降低交易者滑点，交易量带来手续费，又吸引更多 LP，形成网络效应。但 TVL 不是流动性质量的全部，LP 实际持有的是带有手续费收入、库存风险、合约风险和无常损失的风险资产。
