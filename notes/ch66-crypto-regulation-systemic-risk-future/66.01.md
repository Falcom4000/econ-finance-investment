# 66.1 加密资产到底是不是证券

来源：主线参考 Di Maggio《Blockchain, Crypto and DeFi》Ch.11；补充参考本笔记 Ch.5、Ch.21、Ch.22、Ch.57、Ch.64。

加密监管中最核心、也最容易争论的问题，是某个 token 到底是不是证券。这个问题不是文字游戏。若一个 token 被认定为证券，它的发行、交易、托管、披露、投资者保护和市场监管都会进入完全不同的制度框架。若它不是证券，监管重心可能转向商品、支付、反洗钱、消费者保护或税收。

## 证券监管想解决什么问题

证券市场监管的目标并不是阻止风险，而是让风险在更透明、更公平的制度中被承担。股票和债券本来就有风险，投资者可能亏钱，企业可能失败。但监管要求发行人披露信息，交易所监控市场操纵，券商隔离客户资产，审计师检查财务，投资者能在相对清楚的规则下决策。

第 22 章讲股票市场时已经看到，证券是一种对未来现金流和权利的索取。投资者把今天的钱交给发行人，期待未来收益。这种跨时间承诺天然有信息不对称。发行人比投资者更了解项目质量，内部人可能利用信息优势，平台可能挪用客户资产。

所以，判断加密资产是否是证券，背后的问题是：购买者是不是在把资金交给某个项目或团队，并期待他们努力创造收益？如果是，传统证券监管的许多逻辑就会出现。

## Howey Test 的四个问题

美国语境中，原书重点解释 Howey Test。它来自 1946 年美国最高法院案例，用来判断某种安排是否构成投资合同。简化说，若存在资金投入、共同事业、利润期待，并且利润主要依赖他人努力，就可能被视为证券。

把它放到 token 上，可以逐项问：买家是否投入资金？买家的收益是否和一个共同项目或网络绑定？买家是否期待价格上涨或收益分配？这种上涨是否主要来自开发团队、基金会、公司、协议运营者或核心贡献者的努力？

这套测试并不完美，因为它来自传统商业安排，而加密网络可能会演化。一个 token 在早期销售时可能高度依赖团队承诺，后来网络足够去中心化，使用价值更强，证券属性可能变弱。原书称这种状态会“变形”：同一个 token 在不同交易和阶段中，可能被不同看待。

## Bitcoin 为什么通常不被视为证券

Bitcoin 常被认为不属于证券，一个重要原因是它没有传统发行公司，也没有一个对投资者承担经营承诺的中心化管理层。购买 BTC 的人当然可能期待涨价，但这种涨价不主要来自某个公司向持有人承诺经营利润。

这并不意味着 Bitcoin 没有监管。它可能涉及商品交易、反洗钱、税收、托管、交易所和市场操纵监管。非证券不是无监管，只是监管工具不同。

Bitcoin 的例子说明，去中心化程度、发行方式和权利结构很重要。若一个 token 的价值主要来自开放网络本身，而不是某个团队的持续经营承诺，它更难被放进传统证券框架。

## Ethereum 和其他 token 的难点

Ethereum 的情况更复杂。早期 ETH 有融资和项目开发特征；后来以太坊成为开放网络，许多开发者、验证者、应用和用户共同参与。PoS 后，质押收益、验证者角色和生态发展又让监管讨论更复杂。

许多应用 token 更难判断。治理 token 可能没有明确利润分配，却赋予参数调整、国库使用或费用开关权；平台 token 可能被宣传为实用工具，但购买者主要期待升值；稳定币像短期负债而不是证券；NFT 有时像收藏品，有时又附带会员、收益或投资承诺。

因此，不能只看 token 名称。Utility token 不一定只是工具，governance token 不一定只是治理，NFT 不一定只是艺术品。要看发行方式、宣传材料、购买者预期、权利结构和项目方持续作用。

## 为什么证券标签会改变市场结构

若 token 被视为证券，发行人可能需要注册或适用豁免，提供披露文件，接受持续报告和反欺诈规则。交易平台可能需要成为注册交易所、ATS 或 broker-dealer。托管机构要满足客户资产保护规则。二级交易、跨境销售和 DeFi 交易都会变复杂。

这正是原书强调的矛盾：传统证券市场的制度假设有发行人、交易所、经纪商、托管商和清算体系；许多加密资产却在钱包、DEX、DAO、开源代码和全球用户之间流动。把所有 token 简单塞进传统证券盒子，会出现操作上的错位。

但反过来，完全不适用证券监管也会留下投资者保护空白。许多项目确实向公众融资、宣传收益、依赖团队交付，却不提供传统披露。监管难点在于，既不能让技术形式绕开金融实质，也不能让旧制度无差别压住真实创新。

## 小结

加密资产是否是证券，取决于交易实质而不是 token 名称。Howey Test 关注资金投入、共同事业、利润期待和他人努力。Bitcoin 通常不被视为证券，是因为缺少中心化发行人和经营承诺；许多项目 token 则更接近投资合同，尤其在早期融资和宣传收益时。证券标签会带来披露、交易、托管和投资者保护规则，也会与 DeFi 和全球链上市场产生制度摩擦。更合理的分析应看权利、现金流、依赖关系和风险，而不是只看技术包装。
