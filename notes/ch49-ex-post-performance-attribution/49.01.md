# 49.1 绩效归因的基本问题

来源：Giuseppe A. Paleologo, *The Elements of Quantitative Investing*, Chapter 14, "Performance Attribution"。

投资完成以后，最重要的问题之一不是“赚了还是亏了”，而是“为什么赚、为什么亏”。同样 5% 的收益，可能来自真正的选股能力，也可能来自市场 beta；可能来自行业暴露，也可能来自汇率；可能来自策略信号，也可能来自交易时点；可能是可重复的 skill，也可能只是运气。绩效归因就是把总 PnL 拆成可解释部分，使投资者知道收益和风险到底从哪里来。

绩效归因有三个作用。第一，它是组合经理的现实检验。若一个股票多空经理声称自己靠选股赚钱，但归因显示收益主要来自市场反弹或行业暴露，那么策略理解就有问题。第二，它帮助改进投资流程。若亏损来自某个特定因子暴露，可以调整风险约束；若亏损来自交易成本，可以改进执行；若亏损来自选股错误，则需要回到 alpha 模型。第三，它服务于委托代理关系。投资者给管理人付费，是希望获得某种可解释的能力，而不是无意承担廉价 beta。

基本归因从把时间分成若干区间开始。设第 `t` 期初持仓为 `w_t`，资产收益为 `r_t`。持仓带来的组合收益可以写成：

```text
r_t' w_t
```

如果考虑期内交易，实际 PnL 还包含交易 PnL。于是总 PnL 可以分为：

```text
PnL = sum_t (PnL_t - r_t' w_t)
    + sum_t r_t' w_t
```

第一项是交易 PnL，捕捉日内交易、流动性提供、执行成本或期内调仓带来的收益；第二项是期初持仓在期内资产收益上的 PnL。高频策略可能每天收盘接近空仓，却通过日内买卖赚钱，此时交易 PnL 非常重要。低频组合的主要 PnL 则通常来自持仓暴露。

接下来用因子模型分解资产收益：

```text
r_t = B_t f_t + epsilon_t
```

其中 `B_t` 是因子载荷，`f_t` 是因子收益，`epsilon_t` 是特质收益。组合因子暴露为：

```text
b_t = B_t' w_t
```

于是持仓 PnL 可以分解为：

```text
r_t' w_t = b_t' f_t + epsilon_t' w_t
```

把所有时期相加，总 PnL 的基本归因就是：

```text
PnL =
trading PnL
+ factor PnL
+ idiosyncratic PnL
```

其中 factor PnL 还可以逐个因子拆开：

```text
factor PnL_j = sum_t [b_t]_j [f_t]_j
```

这使得投资者可以看到收益究竟来自市场、行业、国家、风格，还是残差选股。

这个分解看似简单，却包含重要的投资哲学。因子 PnL 不一定是坏的，特质 PnL 也不一定是好的。若策略目标就是承担价值或动量风险溢价，那么因子 PnL 是策略核心；若策略收费依据是纯选股能力，那么过多因子 PnL 可能说明组合没有提供足够独特价值。绩效归因必须结合投资授权和策略目标解释，不能机械地把 factor PnL 视为“低级”，把 idiosyncratic PnL 视为“高级”。

归因还必须和风险归因一起看。某个因子贡献了 30% 的 PnL，但如果它贡献了 80% 的风险，说明收益效率可能并不高；某个选股残差贡献不大，但风险占比也很小，可能仍然有不错的 IR。投资中真正关心的是每一类风险承担是否得到合理补偿，而不是某一类 PnL 的绝对大小。

因此，绩效归因是全书前面内容的事后版本。因子模型在事前用于预测风险，在事后用于解释收益；组合优化在事前决定暴露，在事后检查这些暴露是否带来预期贡献；交易成本模型在事前约束换手，在事后解释执行损耗。一个完整的量化投资流程必须让事前模型和事后归因互相校验。

## 小结

绩效归因试图回答收益和亏损来自哪里。基本分解把总 PnL 拆成交易 PnL、因子 PnL 和特质 PnL，并进一步把因子 PnL 分解到具体因子。归因的意义不只是解释历史，更是检验投资流程、识别可重复能力和维护投资者与管理人之间的委托关系。
